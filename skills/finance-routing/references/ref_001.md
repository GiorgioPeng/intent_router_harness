# AG_TRANS 槽位与交接细则

本 reference 只用于补充转账意图 `AG_TRANS` 的详细槽位行为。

## 必填槽位

- `payee_name` 是收款人姓名。当前活跃任务为 `AG_TRANS` 且该槽位缺失时，可以接受短人名式回复。
- `amount` 是转账金额。除非未来 skill 明确引入独立币种槽位，否则保存为不带币种单位的数字字符串。

## 等待补槽行为

- 如果两个必填槽位都缺失，用一句简洁问题同时询问收款人和金额。
- 如果只缺少 `payee_name`，只询问收款人。
- 如果只缺少 `amount`，只询问金额。
- 必须保留 session memory 中已有的已知槽位。
- 多个转账任务串行补槽时，只询问当前 `current_task` 缺失的槽位。
- 如果后续等待任务已有收款人但缺金额，不要在当前任务的补槽问题中混合询问后续任务。

## Router-Only 交接

- 在 `router_only` 模式下，两个必填槽位都齐全时，返回 `ready_for_dispatch`。
- 交接输出必须包含 `ishandover=true`。
- 交接原因必须是 `router_only_ready_for_dispatch`。
