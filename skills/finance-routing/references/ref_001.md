# AG_TRANS Slot and Handoff Detail

This reference is only for detailed transfer slot behavior.

## Required Slots

- `payee_name` is the recipient name. Accept person-like short replies when the current active task is `AG_TRANS` and the slot is missing.
- `amount` is the transfer amount. Store it as a numeric string without currency units unless a dedicated currency slot is introduced by a future skill.

## Waiting Behavior

- If both required slots are missing, ask for both recipient and amount in one concise message.
- If only `payee_name` is missing, ask only for the recipient.
- If only `amount` is missing, ask only for the amount.
- Preserve known slots from session memory.

## Router-Only Handoff

- In `router_only`, when both required slots are present, return `ready_for_dispatch`.
- The handoff output must include `ishandover=true`.
- The handoff reason must be `router_only_ready_for_dispatch`.
