# AG_TRANS 槽位与交接细则

本 reference 只用于补充转账意图 `AG_TRANS` 的详细槽位行为。

## 必填槽位

- `payee_name` 是收款人姓名。当前活跃任务为 `AG_TRANS` 且该槽位缺失时，可以接受短人名式回复。
- `amount` 是转账金额。除非未来 skill 明确引入独立币种槽位，否则保存为不带币种单位的数字字符串。

## 等待补槽行为

- 如果两个必填槽位都缺失，用一句简洁问题同时询问收款人和金额。
- 如果只缺少 `payee_name`，只询问收款人。
- 如果只缺少 `amount`，只询问金额。
- 必须保留 session memory 中已有的已知槽位。
- 多个转账任务串行补槽时，只询问当前 `current_task` 缺失的槽位。
- 如果后续等待任务已有收款人但缺金额，不要在当前任务的补槽问题中混合询问后续任务。
- 当前等待中的 `AG_TRANS` 同时缺少 `payee_name` 和 `amount` 时，如果用户同句给出明确收款人实体和明确金额表达，必须一次性补齐两个槽位，并在 `router_only` 模式下进入 `ready_for_dispatch`。
- 如果用户先给出收款人实体，后半句用“他/她/对方/其/这个人/该收款人”等指代词连接转账、打款或汇款动作和金额，应把该实体作为 `payee_name`，把数量作为 `amount`，例如“收款人甲，给他转1000”“某公司，给对方汇款2000”。
- 示例仅说明结构，不限定收款人名称或金额形式；任意姓名、称呼、实体名与任意数字、中文数字、带币种单位的金额表达都按同一槽位语义处理。
- 中文金额表达需要归一化为数字字符串；例如“三百元”“叁佰圆”“壹仟贰佰元”分别写为 `300`、`300`、`1200`。
- 多任务补槽中，顺序表达用于指向 `task_list` 中对应任务；“第一次给100元”“第一笔100元”应补到第一个任务的 `amount`，“第二次给200元”“第二笔200元”应补到第二个任务的 `amount`。

## Router-Only 交接

- 在 `router_only` 模式下，两个必填槽位都齐全时，返回 `ready_for_dispatch`。
- 当前版本暂不把 `ishandover` 作为核心判断逻辑；后续交接协议调整后再统一收敛 handover 字段。
