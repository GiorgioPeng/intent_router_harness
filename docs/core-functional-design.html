核心职责分层

Spec

定义 LLM 工作界面：surface、prompt、状态机、输出 JSON 约束、skill binding。

Skill

定义领域规则：意图边界、槽位定义、短答补槽、handover 规则。

Service

负责编排：读 session、渲染 prompt、调 LLM、校验输出、合并状态、推 SSE。

流式协议

帧 作用 核心字段
识别帧 告诉客户端当前识别出的业务意图。 status=running
completion_reason=intent_recognized
intent_code=AG_TRANS
业务帧 告诉客户端当前任务状态：缺槽、槽齐、等待下游或失败。 waiting_user_input
ready_for_dispatch
slot_memory
task_list/current_task
结束帧 SSE 流结束。 event: done
data: [DONE]

设计边界

不做

不使用候选意图输入，不输出 primary/candidates，不在服务层写正则提槽。

坚持

业务规则进入 skill，协议约束进入 spec，LLM 只输出结构化 PlannerOutput。

当前阶段

session 先用内存；服务重启后上下文丢失，这属于当前测试阶段边界。
