请求时序图

1. 接口请求
POST /api/v1/message，默认 stream=true，通过 SSE 返回。
2. 读取上下文
session_store.py 按 sessionId 读取 session 生命周期和 TaskRuntimeState。
3. 渲染 surface
planner.py 调用 PromptHarness.render(surface=task_planning)。
4. 渐进式加载 skill
runtime.py 根据 surface、domain、capabilities 匹配 finance-routing/SKILL.md。
5. LLM 分析
LLM 根据 spec + skill + task runtime 输出 PlannerOutput JSON。
6. 协议适配
assistant_service.py 将 PlannerOutput 转为 AssistantProtocolFrame。
7. SSE 输出
asgi.py 输出 event: message 和 event: done。

planner.py

渲染 prompt，调用 LLM，校验 JSON。

llm.py

OpenAI-compatible client。

助手协议回归用例加载和校验。

提槽边界

位置 职责
finance-routing/SKILL.md 定义 AG_TRANS 的必填槽位、短答补槽、槽位继承、缺槽等待、槽齐派发规则。
finance-router-harness.toml 规定 task_planning 的输出协议和状态机约束。
planner.py 把 task runtime、spec、skill 注入 prompt，并调用 LLM 做判断。
assistant_service.py 只合并 LLM 返回的 slot_memory，不硬编码业务提槽。
